Quick Start

The Trex Sandbox environment consists of a Trex virtual machine and CSR1000v router. Using this simple setup, users can experiment with TRex and learn how the package works. The Sandbox environment is dedicated, meaning that the user has sole access to the resources for his/her use. Initial max reservation time is five days.
This is a simple network designed to showcase TRex. It allows the user to run different traffic types through the CSR router.


TRex Sandbox Design

The Sandbox TRex network consists of a TRex VM and a CSR1000v router. These devices have been preconfigured to demonstrate how TRex can load test the router.

  • Interface Eth2 on TRex Linux VM is tied to TRex Port 0 (TRex Client)
  • Interface Eth3 on TRex Linux VM is tied to TRex Port 1 (TRex Server)
  • The CSR1000v is set up to route traffic from interface Gig1 to Gig2 and vice versa
  • The relevant MAC addresses above are correctly defined in the trex_cfg.yaml file in /ect/
  • TRex does not respond to ARP requests. ARP is set up manually on the CSR1000v
Accessing the Sandbox Devices

To access the TRex and CSR devices, hover over the resource icon and select “more options”. Then select “SSH”. A new window will open with command line access.

Credential information is not reqired.

Accessing the Sandbox Devices

To quickly start running some test traffic, perform the following:

  • Login to the TRex server
  • cd v1.90
  • su (password is trex)
  • ./t-rex-64 -f cap2/http_simple.yaml -m 99 -d 60 -l 100
  • Traffic will start to run for 60 seconds. Look at printouts for load information
  • For the above command, TRex uses the .pcap file and parameters defined in the cap2/http_simple.yaml file. Other .yaml files can be called when initiating a traffic run.
  • Ctrl+C to stop traffic
  • For more information, see the TRex support Link in the Support tab
  • CAUTION: Do not exceed Traffic >500Mbps

Note: Please do not make changes to the TRex server or CSR1000v router. This sandbox is intended for learning purposes. All activity is recorded.
